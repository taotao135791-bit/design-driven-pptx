# Work Report / Performance Review

> Applicable scenarios: Work reports, project summaries, performance reviews, quarterly/annual reviews, government/party-style reports, performance defense presentations, etc.
> Style anchor: Internal corporate management reports, emphasizing clear organization, data-rich content, conclusion-first approach, and fact-driven narrative

## Design Philosophy

- **Conclusion first**: Each page leads with conclusions and results, then elaborates on process and details; leadership should grasp core information within 3 seconds
- **Fact-driven**: Data, facts, and objective outcomes form the narrative backbone; avoid subjective feelings and vague embellishments
- **Clear organization**: Modular structure with one topic per page and a clear, traceable logical chain
- **Pragmatic and concise**: Do not pursue visual showmanship; clear information delivery is the top priority, with decoration subordinate to content
- **Data visualization first**: Use charts instead of text when possible; use SmartArt instead of plain lists when applicable

## Information Density: Medium-High (70-80%)

- Each page focuses on one clear topic, content area fill rate 70-80%
- Key achievement data displayed as big numbers (34px+), accompanied by brief comparison notes (YoY/QoQ/completion rate)
- Use lists and tables to organize information; each bullet group has 3-5 items; avoid long text paragraphs
- Moderate whitespace for breathing room, but avoid large blank areas
- Baseline requirement: Header navigation + conclusion-style title + 2-3 KPI big numbers + 4-5 achievement bullets + at least 1 chart or SmartArt + next steps

> **Principle: When leadership flips through, they can grasp each page's core information within 3 seconds; when reading in depth, they can find supporting data and details**

## Text-to-Visual Ratio: Text-Dominant, Charts Reinforce

- Recommended ratio for content pages: approximately 60% text/tables + 30% charts/SmartArt + 10% icons/decoration
- Data achievement pages must include chart visualizations (bar charts, line charts, pie charts, etc.); plain number listings alone are not acceptable
- Process/mechanism/org structure pages should prioritize SmartArt (flowcharts, hierarchy diagrams, cycle diagrams, relationship diagrams) over plain text lists
- Timeline and milestone pages use shape+text timeline layouts
- Minimal photos; focus on infographics and data charts; purely decorative images are prohibited

### SmartArt Usage Recommendations

SmartArt is implemented in PPTD through shape + text + line combinations. SmartArt is a core tool for enhancing information expression in work reports and should be actively used:

- **Flowcharts**: Suitable for work progress workflows, approval processes, project phase displays (e.g., "Requirements Analysis - Solution Design - Implementation & Deployment - Acceptance & Delivery")
- **Hierarchy diagrams**: Suitable for organizational structures, reporting relationships, management system displays
- **Cycle diagrams**: Suitable for closed-loop management mechanisms, PDCA cycles, continuous improvement processes
- **Relationship diagrams**: Suitable for department collaboration relationships, stakeholder analysis, SWOT analysis, core competency models, and other multi-dimensional relationship displays
- **Matrix diagrams**: Suitable for priority ranking, risk assessment (impact/probability matrix), SWOT analysis
- **Funnel diagrams**: Suitable for sales conversion, customer screening, process convergence, and similar scenarios
  * Common chart types not defined in pptd.md (such as waterfall charts) can be built using shape + text box combinations

> **Principle: For any information that can be structured, prefer SmartArt presentation to reduce plain text accumulation**

### Data Chart Usage Recommendations

Charts are the best expression tool for quantified results and should be used extensively:

- **Bar/column charts**: Comparative data (planned vs. actual, department comparisons, quarterly comparisons)
- **Line charts**: Trend data (monthly/quarterly trends, growth curves)
- **Pie/donut charts**: Composition data (revenue structure, budget allocation, issue category proportions)
- **Combo charts** (bar + line): Simultaneously displaying absolute values and rates of change
- **Stacked bar charts**: Showing totals and their internal composition changes
- Every chart must have clear titles, data labels, and legends, ensuring comprehension without verbal explanation

## Color Guidance

### General Corporate Scenarios

- **Steady, professional color schemes** that convey professionalism and credibility; avoid overly playful or casual color tendencies
- If the content involves a brand, prioritize using brand colors; brand colors can be confirmed through search
- Clear color hierarchy: Background, body text, titles, and accents should have distinct visual levels; avoid blurring primary and secondary elements
- Accent color should be used sparingly, only for data highlights and emphasis marking, serving as a visual focal point
- **Strictly prohibited**: Overly vivid multi-color combinations, gradient abuse, neon colors

### State-Owned Enterprise / Government & Party Scenarios

- Primary color system built around corporate brand/theme colors (state-owned enterprises) or China Red, Party Flag Red (government/party scenarios) as the core; gold can serve as an accent color
- Cover/chapter pages may use red gradients or large red color blocks; content pages should maintain white/light gray backgrounds for readability
- Titles and key data highlighted in red or gold; body text remains dark gray/black
- Party emblem, national emblem, and other political symbols should only be used when clearly needed, placed on the cover or fixed decorative positions; they must not be scattered randomly
- Overall tone should be solemn and grand; avoid flashiness

## Font Guidance

- **Titles**: Sans-serif Bold (e.g., MiSans Bold), clean and powerful
- **Body text**: Sans-serif Regular (e.g., MiSans), ensuring screen readability
- Font size hierarchy: Cover title 40-52px - Page title 26-32px - Subtitle 22-26px - Body text 18-22px (use 22px when page text content is light, 20px for moderate, 18px for heavy; must not go below 18px) - Navigation 16-18px - Footnotes 12-16px
- KPI big numbers 38px, using accent color to stand out
- Consistently use sans-serif throughout; no serif/sans-serif contrast needed
- For state-owned enterprise scenarios, titles may optionally use siyuanSongti, alimamadongfangdakai, or other government-style fonts

## Content Pages

Content pages may use the following elements, but none are mandatory:
- **Navigation bar**: Recommend setting up a horizontal or vertical navigation bar with width/height matching the page width/height, chapter titles evenly distributed, current chapter highlighted with a white rectangle with themed border and readable text color, helping the audience track presentation progress and overall structure
- **Page title**: Conclusion-style achievement sentence (one line), summarizing the page's core result rather than a generic topic word
- **Company logo**: When the user specifies a company name, it is recommended to search online for the company logo (or extract from user-uploaded attachments) and place it at a consistent fixed position on every page to clearly indicate the reporting unit
- A standard structure of **Navigation bar - Main title - Divider line - Charts/SmartArt - KPI big numbers + achievement bullets - Report info/page number** can be adopted, but this is only one possible structure and is not mandatory
- **Content area** common elements:
  - Charts/SmartArt, quickly structuring complex information into easily digestible content
  - KPI big numbers, prominently highlighting important information
  - Achievement bullets with medium-length sentences explaining core viewpoints and content, neither too brief nor too lengthy

### Performance Defense Scenario Special Requirements

Performance defense presentations differ from regular work reports and require special attention:

- **Personal profile page**: Opening should include basic personal information, job responsibilities, and tenure period, presented concisely
- **Self-assessment should be objectively quantified**: Support capability evaluations with data; avoid hollow phrases like "diligent and responsible" or "proactive and enthusiastic"
- **Clear benchmarking dimensions**: Address each dimension systematically — position KPIs, responsibility fulfillment, innovation breakthroughs, team contributions
- **Growth and shortcomings coexist**: Recommend including a "Shortcomings & Improvement" section to demonstrate self-awareness, but keep it under 15% of total content
- **Specific future plans**: Next steps should be linked to position goals with clear quantified targets and timelines
- **Recommended defense structure**: Personal introduction - Job responsibilities - Work achievements (broken down by dimension) - Highlights & innovation - Shortcomings & improvement - Future plans

### State-Owned Enterprise Reporting Scenario Special Requirements

- **Political positioning first**: The first page or chapter should reflect policy background, directives from higher authorities, and implementation of strategic deployments
- **Standardized reporting language**: Use formal expressions such as "under the guidance of...," "earnestly implementing...," "steadily advancing...," "achieving significant results," etc.; avoid colloquial language
- **Rigorous data**: All data must be traceable; financial data must specify the basis (e.g., tax-inclusive/exclusive, consolidated/standalone); external industry or benchmark data citations should use `<a href="url">` hyperlinks to indicate sources
- **Benchmark alignment**: Work achievements must be compared item-by-item against annual targets and assessment indicators from higher authorities, using comparison tables or completion rate charts
- **Party building integration**: If party building work is involved, a dedicated chapter should be included, reflecting the logic of "party building leading business operations"
- **Standardized chapter naming**: Recommend using standard chapter names such as "Work Review," "Key Achievements," "Experience & Best Practices," "Existing Issues," "Next Steps"

## Narrative Style: Achievement Reporting

- Each page title summarizes the core conclusion or achievement (e.g., "Q3 Sales Revenue Up 23% YoY, Exceeding Target"); generic titles like "Work Achievements" with no information value are prohibited
- Content organized following the **Achievement - Process - Next Steps** logic
- Let data speak: Quantify results with numbers wherever possible; avoid vague qualitative descriptions (e.g., replace "significant improvement" with "improved by 23%")
- Problems and challenges sections must include solutions or action plans; presenting problems without solutions is not acceptable
- Avoid excessive modesty or excessive self-promotion; maintain an objective tone

## Content Expression Techniques

- **KPI big numbers**: Core achievement metrics displayed as large numbers + comparison benchmarks (YoY/QoQ/completion rate/industry average), giving numbers a frame of reference
- **Progress bars/completion rates**: Use shapes to simulate progress bars, intuitively showing target achievement; annotate with percentage values
- **Timelines**: Project milestones and work progress rhythm displayed using timeline SmartArt or shape combination layouts
- **Comparison tables**: Planned vs. actual, current period vs. previous period, target vs. completed — clear table comparisons with achievement rates annotated
- **Bullet points**: Each bullet starts with a bolded keyword, followed by specific data and quantified results
- **Next steps**: Clear action items + responsible persons + time nodes; recommend using tables or SmartArt for presentation
- **SmartArt process/relationship diagrams**: Work processes, organizational relationships, management mechanisms, and other structured information should prioritize graphical expression
- **Data charts**: Trend, comparison, and composition data must use charts; text-only descriptions are not acceptable

## Decoration Prohibitions

| Prohibited | Alternative |
|--------|---------|
| Large decorative images | Data charts and SmartArt |
| Too many colors (>4) | Primary + secondary + accent, three colors as the main palette |
| WordArt/text effects | Standard font with bold is sufficient |
| Meaningless icon overload | Use icons only when visual differentiation is needed |
| Excessive visual effects | Clean, restrained static display |
| Purely decorative background patterns | Solid colors or very light textures |
| Hollow qualitative descriptions | Quantified data + chart evidence |
| Rounded rectangles | Prefer sharp-cornered rectangles or borderless layouts to maintain a pragmatic and concise business style |
